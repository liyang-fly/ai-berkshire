# 本地信息源录入：投资研究技能的逆向写入

对 $ARGUMENTS 进行本地信息源录入和管理。

## 设计原理

本 Skill 是 `/investment-research` 和 `/industry-research` 的**逆向写入**。

两个研究技能按以下维度读取信息，因此本地信息必须在写入时标注**目标分析维度**，确保研究技能执行时能精确匹配：

```
写入时标注维度 → 索引中可检索 → 研究技能按维度拉取 → 填入对应分析模块
```

## 实体类型

| 实体类型 | 示例 | 文件夹 | 消费技能 |
|---------|------|--------|---------|
| 公司 | 腾讯、茅台、英伟达 | `data/local-notes/腾讯/` | investment-research |
| 行业 | AI、白酒、新能源 | `data/local-notes/AI行业/` | industry-research |
| 主题 | 宏观利率、地缘政治 | `data/local-notes/宏观利率/` | 两者 |

## 支持的操作

### 1. 录入新笔记

**输入格式**：`实体名：主题：内容`

例如：
- `腾讯：微信支付线下体验：今天在3家便利店用微信支付...`
- `AI行业：大模型降价战观察：最近几家大模型厂商都在降价...`
- `茅台：飞天终端价调研：跑了几家烟酒店，实际成交价...`

**执行流程**：

1. 解析输入，提取实体名、主题、内容
2. 自动判断实体类型（公司/行业/主题），如果无法判断则询问用户
3. **关键步骤：判断目标分析维度**。根据内容自动判断这段信息应该被哪个研究模块消费，如果无法判断则询问用户，列出可选维度
4. 生成文件名：`{YYYYMMDD}-{主题}.md`
5. 写入笔记，使用标准模板
6. 更新索引

#### 笔记模板

```markdown
# {主题}

**日期**：{YYYYMMDD}
**实体**：{实体名}
**实体类型**：{公司 | 行业 | 主题}
**分析维度**：{见下方维度表，可多选}
**大师框架**：{段永平 | 巴菲特 | 芒格 | 李录 | 通用}
**信息类型**：{一手观察 | 二手分析 | 人脉信息 | 阅读笔记}
**信息源**：{具体来源描述}
**可信度**：{高 | 中 | 低}

---

{内容}

---

**对投资判断的意义**：
**与现有判断一致/矛盾**：
**待验证的点**：
```

#### 分析维度对照表（写入时必选）

写入时根据内容选择对应的分析维度，研究技能执行时按此维度拉取：

| 维度 | 说明 | 消费技能 | 对应模块 |
|------|------|---------|---------|
| **生意本质** | 产品/服务体验、商业模式观察、收入质量判断 | investment-research | 第二步（段永平） |
| **护城河** | 品牌力、转换成本、网络效应、规模效应、技术壁垒 | investment-research | 第三步（巴菲特） |
| **风险** | 潜在失败路径、空方论点、行业风险信号 | investment-research + industry-research | 第四步（芒格）/ 第五步 |
| **管理层** | 创始人/CEO印象、关键决策、企业文化、利益对齐 | investment-research | 第五步（段永平+巴菲特） |
| **行业趋势** | 行业变化、政策方向、技术路线、TAM判断 | investment-research + industry-research | 第六步（李录）/ 第一、二步 |
| **估值** | 终端价格调研、可比交易、隐含预期判断 | investment-research | 第七步（巴菲特+段永平） |
| **财务数据** | 手工记录的关键数字、非公开财务信息 | investment-research | 第一步（数据收集） |
| **竞争格局** | 市场份额变化、竞争对手动态、新进入者 | investment-research + industry-research | 第一步 / 第三步 |
| **投资逻辑链** | 产业链因果关系验证、供需变化信号 | industry-research | 第一步（逻辑链验证） |
| **产业链** | 上下游关系、卡脖子环节、利润分配 | industry-research | 第二步（产业链全景） |
| **综合判断** | 跨维度的整体判断、投资决策相关 | 两者 | 综合决策备忘录 |

#### 信息类型（描述信息性质）

| 类型 | 说明 | 可信度默认值 |
|------|------|------------|
| 一手观察 | 亲眼所见、亲自使用、亲身交流 | 高 |
| 二手分析 | 研报/文章/书籍的摘要批注 | 中 |
| 人脉信息 | 从从业者/供应商/客户获取 | 中-高（取决于关系） |
| 阅读笔记 | 已消化信息的结构化记录 | 中 |

#### 大师框架（这段信息最相关的大师视角）

| 框架 | 关注点 |
|------|--------|
| 段永平 | 生意本质、对的生意+对的人+对的价格 |
| 巴菲特 | 护城河、管理层诚信、内在价值 |
| 芒格 | 逆向思考、风险、跨学科验证 |
| 李录 | 文明趋势、范式转移、长期演进 |
| 通用 | 不特定关联某个大师 |

### 2. 查看已有笔记

**输入格式**：`list {实体名}` 或 `list all` 或 `list 维度:{维度名}`

例如：
- `list 腾讯` — 查看腾讯所有笔记
- `list all` — 查看所有实体及笔记数
- `list 维度:护城河` — 查看所有标注了"护城河"维度的笔记
- `list 维度:管理层` — 查看所有管理层相关笔记

### 3. 搜索笔记

**输入格式**：`search {关键词}`

例如：`search 护城河`、`search 降价`、`search 黄峥`

### 4. 读取指定笔记

**输入格式**：`read {实体名} {文件名}` 或 `read {实体名} latest`

例如：`read 腾讯 latest`、`read AI行业 20260710-大模型降价战`

## 与研究技能的联动机制

### investment-research 读取时

```
第零步执行：
1. 读取 index.json
2. 匹配 entities[公司名] → 按 dimensions 分类拉取笔记
3. 匹配相关行业 entities → 筛选 type="行业" 的实体
4. 将笔记按分析维度填入对应模块：
   - 生意本质 → 填入第二步
   - 护城河 → 填入第三步
   - 风险 → 填入第四步
   - 管理层 → 填入第五步
   - 行业趋势 → 填入第六步
   - 估值 → 填入第七步
   - 财务数据 → 填入第一步
   - 竞争格局 → 填入第一步
```

### industry-research 读取时

```
执行前检索：
1. 读取 index.json
2. 匹配 entities[行业名]
3. 按 dimensions 拉取：
   - 投资逻辑链 → 填入第一步
   - 产业链 → 填入第二步
   - 行业趋势 → 填入第一、六步
   - 风险 → 填入第五步
   - 竞争格局 → 填入第三步
```

## 最佳实践

1. **及时记录**：听完播客、看完访谈、试用产品后立刻记录
2. **维度必选**：写入时想清楚"这段信息在研究框架的哪个模块会被用到"
3. **区分事实与观点**：标注亲眼所见 vs 听说的 vs 推测的
4. **标注可信度**：一手观察 > 可靠人脉 > 公开报道 > 道听途说
5. **投资启示必写**：每篇笔记都问"这对投资判断意味着什么？"
6. **行业与公司分开**：行业趋势写入行业笔记，公司信息写入公司笔记
7. **定期回顾**：每月回顾，标记已过时信息
